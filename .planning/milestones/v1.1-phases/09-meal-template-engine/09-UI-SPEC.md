## Phase UI Scope

**This is a backend engine phase.** No new UI screens, routes, or form components are introduced. The only UI surface that changes is `ruleDescriptions.ts` — the `describeRule()` function must handle the `meal-template` CompiledFilter variant so that any manually-inserted test rules render a readable label in `RuleRow` rather than falling through to an unhandled type.

**All UI form work (MealTemplateFields, RuleForm tab, settings removal) is deferred to Phase 10.**

## Color

All colors are CSS variables from globals.css (oklch color space). Do not hard-code hex values.

| Role | CSS variable | Usage |
|------|-------------|-------|
| Dominant (60%) | `--background` (oklch 1 0 0 / near-white) | Page background, surfaces |
| Secondary (30%) | `--card`, `--muted` (oklch 0.966) | Cards, sidebar, nav, muted backgrounds |
| Accent (10%) | `--primary` (oklch 0.852 0.199 91.936 — olive-yellow) | Toggle switch enabled state, primary buttons |
| Destructive | `--destructive` (oklch 0.577 0.245 27.325 — red) | Delete confirmation alert, destructive button variant only |
| Muted text | `--muted-foreground` (oklch 0.58) | Rule summary text, secondary descriptions |
| Input | `--input` (oklch 0.93) | Toggle switch disabled state |

Accent (`--primary`) is reserved for:
- Toggle switch enabled state (`bg-primary`)
- Primary action buttons (`Button` default variant)

Destructive (`--destructive`) is reserved for:
- Delete confirmation `Alert` with `variant="destructive"`
- Delete `Button` with `variant="destructive"`

Source: RuleRow.tsx, globals.css.

## Copywriting Contract

Phase 9 has no new user-facing copy for forms or screens. The only new copy needed is the `describeRule()` label for `meal-template` rules — used in `RuleRow` summary text.

| Element | Copy | Notes |
|---------|------|-------|
| meal-template rule type badge | `"Meal Template"` | Follows existing pattern: `"No Repeat"` / `"Scheduling"` in RuleRow |
| meal-template rule description (slot assignment only) | `"Rice-based: allowed at lunch, dinner"` | Format: `"{base_type}: allowed at {slots}"`. Use "all slots" when allowed_slots is null. |
| meal-template rule description (excludes only) | `"Bread-based: exclude subzi, sweet extras"` | Format: `"{base_type}: exclude {component_types}{extra_categories}"` |
| meal-template rule description (require only) | `"Bread-based: require liquid extra"` | Format: `"{base_type}: require {category} extra"` |
| meal-template rule description (combined) | `"Rice-based (lunch, dinner): exclude curry, require liquid extra"` | Show slot scope in parens when slots context is present; omit when null |
| meal-template rule description (universal) | `"Bread-based: no subzi"` | When days and slots are both null, no context qualifier |
| Warning (generator constraint relaxed) | No user-visible copy — console warn only | Source: CONTEXT.md D-10; warnings go to `warnings[]` array, surfaced by existing WarningBanner if applicable |
| Empty state (rules list, no rules yet) | Existing copy unchanged — not modified in this phase | Source: RuleEmptyState.tsx |
| Destructive (delete rule) | `"Delete this rule? This cannot be undone."` / `"Delete"` / `"Keep rule"` | Existing RuleRow copy — unchanged |

## Interaction Contract

### Toggle switch (existing — unchanged)

- Visual: 20px × 36px pill; `bg-primary` when enabled, `bg-input` when disabled
- Thumb: 16px × 16px white circle; translates 16px right when enabled
- Role: `role="switch" aria-checked={enabled}`
- No change needed for meal-template rules.

### Delete flow (existing — unchanged)

1. User taps trash icon → inline `Alert` with `variant="destructive"` expands below the rule row.
2. Alert contains: confirmation text + "Delete" button (destructive) + "Keep rule" button (ghost).
3. Confirming delete calls `deleteRule(id)` and collapses the strip.
4. No modal; no navigation.

### RuleRow rendering for meal-template (new in this phase)

- Rule type badge: `"Meal Template"` (secondary badge, same size as existing)
- Summary text: result of `describeRule()` — see Copywriting Contract above
- Toggle and delete: identical behavior to existing rule types
- Disabled state: `text-muted-foreground` applied to the entire row (same class as existing)
