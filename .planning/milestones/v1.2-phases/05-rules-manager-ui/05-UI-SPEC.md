## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline checkbox/label gap (gap-1 = 4px) |
| sm | 8px | Compact element spacing, tag/badge gaps |
| md | 16px | Default element spacing, form field vertical rhythm |
| lg | 24px | Section padding, heading margin-bottom (mb-6) |
| xl | 32px | Layout gaps between form sections |
| 2xl | 48px | Minimum touch target height for rule rows (min-h-[48px]) |
| 3xl | 64px | Page-level top/bottom padding (py-8 = 32px top+bottom) |

Exceptions:
- Rule rows: `min-h-[48px]` — matches established ComponentRow touch target (Source: ComponentRow.tsx)
- Page horizontal padding: `px-4 sm:px-8` — matches established PlanBoard and ComponentLibrary pattern (Source: PlanBoard.tsx, ComponentLibrary.tsx)
- Nav bar: `px-4 py-3 sm:px-8` — do not change; AppNav is shared (Source: AppNav.tsx)

## Color

All values from CSS custom properties defined in globals.css (Source: globals.css).

| Role | CSS Token | OKLCH (light) | Usage |
|------|-----------|---------------|-------|
| Dominant (60%) | --background | oklch(1 0 0) | Page background, form background |
| Secondary (30%) | --card / --muted | card: oklch(1 0 0) / muted: oklch(0.966 0.005 106.5) | Rule row hover state (`hover:bg-muted/40`), form section backgrounds, empty state example cards |
| Accent (10%) | --primary | oklch(0.852 0.199 91.936) — olive-yellow | Toggle switch in enabled state only, Save button, "Add Rule" button |
| Destructive | --destructive | oklch(0.577 0.245 27.325) — red | Delete icon/button only |
| Warning (custom) | amber className override | border-amber-500 bg-amber-50 text-amber-900 | Zero-match warning Alert only (no shadcn warning variant exists) |
| Muted text | --muted-foreground | oklch(0.58 0.031 107.3) | Inactive rule row text, nav links, helper text, greyed-out example rules |
| Input border | --input | oklch(0.93 0.007 106.5) | Form input borders, toggle switch off-state |

**Accent reserved for:** Save button, "Add Rule" button, enabled toggle switch track. NOT used for labels, badges, headings, icons, or nav links.

**Inactive rule rows (D-08):** Apply `text-muted-foreground` to the entire row content (name, summary, badge) when `rule.enabled === false`. Toggle switch uses `bg-input` (grey) in off state.

**Dark mode:** All tokens have dark-mode values declared in globals.css. No additional dark-mode overrides needed for Phase 5.

## Interaction Contracts

### Form Validation — Save button enabled when:

| Rule type | Required fields |
|-----------|----------------|
| Any | `name.trim() !== ''` AND `ruleType` is set |
| day-filter | `days.length > 0` (tag filter fields are optional — all-tag match is valid) |
| no-repeat | `component_type !== ''` |
| require-component | `component_id !== null` AND `days.length > 0` |

Source: RESEARCH.md Pitfall 2.

### Impact Preview — per rule type:

| Rule type | Preview text |
|-----------|-------------|
| day-filter | "This rule affects N of {total} components." Where N = count of ComponentRecords matching the TagFilter using `.every()` AND predicate |
| no-repeat | "Ensures no {component_type} repeats within a week." |
| require-component | "This rule will require [component name] on [N day(s)]." |

Zero-match condition applies only to `day-filter`. Show amber warning when `impactCount === 0` AND `ruleType === 'day-filter'`. Save remains enabled.

### Delete confirmation (Claude's discretion — resolved):

Use inline `DeleteConfirmStrip` (same as ComponentRow), expanded below the row. "Delete?" text + "Confirm" (destructive) + "Cancel" (ghost). No modal. This matches the established library pattern.

### Form type switching:

Switching rule type via Tabs dispatches `SET_RULE_TYPE` action in `useReducer`, resetting ALL type-specific fields to defaults atomically. This prevents stale ghost state (Source: RESEARCH.md Pitfall 1).

### Empty state example clicks:

Clicking a greyed-out example rule navigates to `/rules/new?preset={preset-id}`. The form page reads `useSearchParams()` on mount and initializes the reducer with preset values. Three presets:

| Preset id | Pre-filled values |
|-----------|------------------|
| fish-fridays | name: "Fish Fridays", ruleType: day-filter, days: [friday], filter: { protein_tag: fish } |
| no-repeat-subzi | name: "No repeat subzi", ruleType: no-repeat, component_type: subzi |
| weekend-special | name: "Weekend special", ruleType: day-filter, days: [saturday, sunday], filter: { occasion_tag: weekend } |

### AppNav update:

Add `<Link href="/rules" className="text-sm text-muted-foreground hover:text-foreground">Rules</Link>` between the "Meal Library" and "Slot Settings" links. Same styling as existing nav links.

## States & Variations

| State | Component | Visual Treatment |
|-------|-----------|-----------------|
| Rule enabled | RuleRow | Normal foreground color; toggle track = bg-primary |
| Rule disabled | RuleRow | Entire row text = text-muted-foreground; toggle track = bg-input |
| Save button disabled | Button | opacity-50 cursor-not-allowed (default Tailwind/shadcn disabled state) |
| Zero-match warning | RuleImpactPreview | Amber Alert: border-amber-500 bg-amber-50 text-amber-900 (dark: bg-amber-950 text-amber-200 border-amber-700) |
| Form field invalid (empty required) | Input / Select / Tabs | No explicit error styling until Save is attempted; Save button simply stays disabled |
| List loading | RuleList | Render nothing (useLiveQuery returns undefined → early return null) |
| Empty list | RuleList | RuleEmptyState with 3 greyed-out example cards |
| Example rule card | Empty state | opacity-50 or text-muted-foreground; cursor-pointer; hover lifts opacity to 80% |

## Accessibility Contract

| Pattern | Requirement |
|---------|------------|
| Toggle switch | `role="switch"` + `aria-checked={rule.enabled}` on the `<button>` element |
| Delete icon button | Tooltip provides accessible label; also add `aria-label="Delete rule"` on the button |
| Form | All inputs have associated `<Label>` via `htmlFor`/`id` |
| Checkbox group (days) | Wrap in `<fieldset>` + `<legend>Days</legend>` |
| Tabs (rule type selector) | Uses Radix Tabs — keyboard navigation (arrow keys) built in |
| Color contrast | All text uses --foreground or --muted-foreground on --background; amber warning uses dark-enough text-amber-900 for WCAG AA |
