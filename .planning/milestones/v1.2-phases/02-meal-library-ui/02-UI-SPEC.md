## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, tag chip inner padding, inline gaps between dietary badges |
| sm | 8px | Row internal padding (between icon and label), chip-to-chip gap |
| md | 16px | Card/row horizontal padding, input field internal padding, form field gap |
| lg | 24px | Tab content top padding, section gap between search bar and list |
| xl | 32px | Page horizontal gutter on desktop, gap between settings grid and exception section |
| 2xl | 48px | Empty state vertical centering offset |
| 3xl | 64px | Page-level top/bottom padding |

Exceptions:
- Touch targets for icon-only buttons (delete, collapse row): minimum 44px hit area regardless of visual size — achieved via padding, not by changing the visual element size.

Source: 8-point scale standard; exceptions from WCAG 2.5.5 minimum target size.

## Color

shadcn preset `bgAUzxKUy` establishes the concrete token values. Map them to the 60/30/10 contract as follows:

| Role | shadcn Token | Usage |
|------|-------------|-------|
| Dominant (60%) | `background` | Page background, tab content area, expanded row background |
| Secondary (30%) | `card` / `muted` | Collapsed row surface, slot assignment grid cell, settings panel background |
| Accent (10%) | `primary` | Active tab indicator, "Add Component" button, selected tag chip filled state, checkbox checked fill in slot grid |
| Destructive | `destructive` | Delete component button icon + label, delete confirmation inline text |

Accent (`primary`) is reserved for **exactly these elements**:
1. Active tab underline/indicator
2. "Add [ComponentType]" button (filled)
3. Selected/active tag filter chip (filled background)
4. Checkbox checked state in the slot assignment grid

Accent is NOT used on: row hover states, search input borders, section headers, table grid lines, or neutral informational badges.

Destructive is used ONLY for the delete action control and the inline confirmation warning text. No other UI element uses destructive color.

Source: shadcn preset provides exact hex values; roles derived from CONTEXT.md layout decisions and standard 60/30/10 allocation.

## Focal Point

The primary focal point on the component library screen is the **tab bar + search input pair** at the top of the content area. This row anchors the user's eye immediately on arrival: the active tab communicates which component type is in view, and the search input invites immediate filtering. The list below it is secondary; the empty state is tertiary. No decorative hero imagery or illustration is used — the focal point is purely structural.

## Interaction Contracts

### Tab navigation
- Four tabs rendered as a horizontal tab bar at the top of the component library page: **Bases | Curries | Subzis | Extras**.
- Each tab maintains its own independent search query and active tag filter state — switching tabs does not reset another tab's filters.
- Default active tab on page load: Bases.

### Collapsed row (list view)
- Each component renders as a single row: `[Component name] [dietary badge] [regional badge] [compatible_base_types badge — Extras only] [delete icon button]`.
- Row height: 48px.
- Clicking anywhere on the row except the delete button expands it inline.
- Only one row can be expanded at a time — opening a new row collapses the previously open one.

### Delete icon button visibility
- The delete button is **icon-only on all viewports** (trash icon, lucide-react `Trash2`). A visible text label is not shown on any breakpoint — the row is too dense to accommodate it.
- **Desktop:** A tooltip ("Delete [name]") appears on hover via shadcn `tooltip`. The tooltip is the sole affordance for users who are not familiar with the trash icon.
- **Mobile:** No hover state. The `aria-label="Delete [name]"` attribute covers screen-reader accessibility. The 44px minimum touch target (achieved via padding) covers motor accessibility.

### Expanded row (inline editor)
- Expands below the collapsed row in the same list position without navigation.
- Fields shown depend on `componentType`:
  - **Base:** name (text), base_type (select: rice-based / bread-based / other), dietary tags (checkbox group), protein tags (checkbox group), regional tags (checkbox group), occasion tags (checkbox group).
  - **Curry:** name (text), dietary tags, protein tags, regional tags, occasion tags. No base_type field.
  - **Subzi:** same as Curry.
  - **Extra:** name (text), extra_category (select: Liquid / Crunchy / Condiment / Dairy / Sweet), compatible_base_types (checkbox group: rice-based / bread-based / other), dietary tags, regional tags, occasion tags.
- Action buttons: `[ Save [ComponentType] ]` (primary, filled — label is dynamic, e.g. "Save Base", "Save Curry", "Save Subzi", "Save Extra") and `[ Discard changes ]` (ghost) side by side at the bottom of the expanded form.
- "Save [ComponentType]" calls `updateComponent`. "Discard changes" collapses the row discarding all edits.

### Search and filter
- Text search input at the top of each tab content area, below the tab bar.
- Search filters by component name (case-insensitive, partial match).
- Tag filter chips rendered as a horizontal scrollable row below the search input: one chip per available dietary value + one per regional value.
- Chips toggle on/off. Multiple chips can be active simultaneously — results satisfy ALL active chips (AND logic).
- Chips with no matching components in the current list are visible but disabled (muted color, non-interactive) to signal "no results for this filter."

### Add component
- `[ + Add {ComponentType} ]` button pinned at the bottom of each tab's list.
- Clicking appends a new blank expanded-row form at the bottom of the list (same inline form as the edit form, but fields are empty).
- Save calls `addComponent`. Cancel removes the blank form without creating a record.

### Delete component
- Delete icon button (trash icon, 44px hit target, aria-label: "Delete [name]") on the far right of each collapsed row.
- Clicking the delete button does NOT immediately delete. It reveals an inline confirmation strip below the row: "Delete [name]? This cannot be undone. [ Delete ] [ Keep [name] ]".
- "Delete" in the confirmation strip calls `deleteComponent` and removes the row.
- "Keep [name]" collapses the confirmation strip and the row returns to normal state.
- If the component has been referenced in any saved plan (future-phase concern), the inline strip additionally shows: "This component is used in saved plans. Deleting it will remove it from those plans."

### Slot assignment settings screen
- Accessible via a "Slot Settings" link/button in the page header or app navigation.
- Renders as a dedicated full-width settings page (not a drawer or modal).
- **Checkbox grid:** Rows = base types (rice-based, bread-based, other). Columns = Breakfast, Lunch, Dinner. Each cell is an independent checkbox. Multiple columns can be checked per row.
- **Seed defaults pre-checked on first load:**
  - other → Breakfast ✓, Lunch ✗, Dinner ✓
  - rice-based → Breakfast ✗, Lunch ✓, Dinner ✗
  - bread-based → Breakfast ✗, Lunch ✗, Dinner ✓
- **Component exceptions section:** Collapsed by default, labelled "Component exceptions (optional)". Expands to show a list of per-component slot override entries. An "Add exception" button opens an inline form: component search/select + checkbox group (Breakfast / Lunch / Dinner).
- **Poori exception pre-seeded:** ComponentRecord for Poori gets an entry in `UserPreferences.component_slot_overrides` pointing to Breakfast only (overriding the `other` base type's Breakfast + Dinner default).
- Save button calls `putPreferences`. Changes take effect immediately (no confirmation needed — non-destructive).

### Empty states
- Shown per tab when a tab has zero components (before seed loads, or if all are deleted).
- Shown per tab when active search/filter returns zero matches.
