  <behavior>
    **PLAN-01 — Full 21-slot generation:**
    - generate() returns 21 PlanSlot entries (7 days x 3 meal slots)
    - Each slot has day (monday-sunday) and meal_slot (breakfast/lunch/dinner)
    - Each slot has a base_id (required), optional curry_id, optional subzi_id, extra_ids array
    - Slot fill order: all 7 breakfasts first, then 7 lunches, then 7 dinners

    **PLAN-04 — Extra compatibility:**
    - Extras only assigned when component's compatible_base_types includes the slot's selected base type
    - Rasam (compatible_base_types: ['rice-based']) never appears with a bread-based base
    - Extra count per slot respects UserPreferences.extra_quantity_limits[meal_slot]
    - Mandatory extras: when base_type_rules says 'other' requires 'condiment', at least one condiment extra is force-added

    **RULE-03 — Day-based rules:**
    - DayFilterRule with days: ['friday'] and filter: { protein_tag: 'fish' } only affects Friday slots
    - DayFilterRule with slots: ['breakfast'] only affects breakfast on those days
    - DayFilterRule with slots: null affects all three meal slots on those days
    - Tag filter AND logic: all specified tags must match on the component

    **RULE-04 — No-repeat rules:**
    - NoRepeatRule with component_type: 'subzi' prevents same subzi from appearing in two different slots within the week
    - Three separate tracking Sets: usedBaseIds, usedCurryIds, usedSubziIds
    - No-repeat applies within a slot type only (all 7 breakfasts share no-repeat tracking)
    - Actually per CONTEXT.md: no-repeat tracks across ALL slots in the week (not per slot type)

    **Frequency weighting:**
    - frequent components (weight 3) appear roughly 3x more than normal (weight 1)
    - rare components (weight 0.3) appear roughly 0.3x as often as normal
    - component.frequency defaults to 'normal' when undefined

    **Recency halving:**
    - First use of a component: weight multiplier = 1 (0.5^0)
    - Second use: multiplier = 0.5 (0.5^1)
    - Third use: multiplier = 0.25 (0.5^2)
    - effective_weight = frequency_weight * 0.5^(times_used_so_far_this_week)

    **Over-constrained handling:**
    - When no candidates satisfy all active rules, pick from unfiltered pool
    - Emit a Warning for each relaxed rule
    - Slot restrictions are HARD constraints (never relaxed)
    - When zero eligible bases exist for a slot (empty base pool after slot restrictions), emit warning with rule_id: null

    **Performance:**
    - Full 21-slot generation completes in under 500ms
  </behavior>

**Generator implementation (src/services/generator.ts):**

The generator function signature:
```typescript
export async function generate(): Promise<GeneratorResult>
```

Internal structure:
1. Load data: `Promise.all([getAllComponents(), getPreferences(), getEnabledRules()])`
2. Validate rules with `CompiledFilterSchema.safeParse()` — skip invalid rules with warning
3. Partition components by type: bases, curries, subzis, extras (from allComponents)
4. Initialize tracking state: `usageCount = new Map<number, number>()`, `usedBaseIds/usedCurryIds/usedSubziIds = new Set<number>()`
5. Fill order: `['breakfast', 'lunch', 'dinner']` outer, `ALL_DAYS` inner
6. For each (meal_slot, day):
   a. Filter bases by slot eligibility (hard constraint from preferences)
   b. Apply applicable rules to base pool (soft constraints)
   c. Pick base via weighted random with frequency * recency
   d. Filter curries by applicable rules, pick via weighted random
   e. Filter subzis by applicable rules + no-repeat set, pick via weighted random
   f. Filter extras by compatible_base_types of selected base, respect extra_quantity_limits
   g. Check mandatory extra enforcement (base_type_rules)
   h. Record usage counts for recency tracking, add to no-repeat sets
   i. Create PlanSlot and push to slots array
7. Return `{ plan: { slots }, warnings }`

Key helper functions to implement:
- `weightedRandom<T>(items: T[], getWeight: (item: T) => number): T` — cumulative probability selection
- `effectiveWeight(component: ComponentRecord, usageCount: Map<number, number>): number` — frequency_weight * 0.5^usage
- `isRuleApplicable(rule: CompiledFilter, day: DayOfWeek, slot: MealSlot): boolean` — checks day/slot targeting
- `matchesTagFilter(component: ComponentRecord, filter: TagFilter): boolean` — AND logic on specified tags
- `getEligibleBases(slot: MealSlot, prefs: UserPreferencesRecord, bases: ComponentRecord[]): ComponentRecord[]` — slot restriction filter

Important: Use `getAllComponents`, `getPreferences`, `getEnabledRules` from `@/services/food-db` — do NOT import `db` directly.
