# Quick Task 260326-w99: Meal Template Flexible Selector (Base/Tag/Component) Summary

**One-liner:** Replaced fixed base-type radio in meal-template rules with a three-mode selector discriminated union (base/tag/component), updating schema, Dexie migration, compiler, descriptions, form UI, and generator engine.

## What Was Built

### Schema Layer (`src/types/plan.ts`)
- Replaced `base_type: z.enum(...)` with `selector: z.discriminatedUnion('mode', [...])` in `CompiledFilterSchema`
- Three selector modes: `base` (with `base_type`), `tag` (with `filter: TagFilterSchema`), `component` (with `component_id: number`)
- Updated `RuleDefinition` meal-template variant to use the same selector shape

### Database Migration (`src/db/client.ts`)
- Added `migrateMealTemplateSelector(cf: unknown): unknown` — pure function converts old `base_type` field to `selector: { mode: 'base', base_type }`
- Added Dexie v8 upgrade that applies the migration to all stored rules
- Note: Dexie v7 still creates rules with legacy `base_type` (from the prefs migration); v8 immediately upgrades them

### Rule Compiler (`src/services/rule-compiler.ts`)
- `compileRule` meal-template case now passes `selector: def.selector` instead of `base_type: def.base_type`

### Rule Descriptions (`src/components/rules/ruleDescriptions.ts`)
- `describeRule` meal-template case renders:
  - `mode === 'base'` → `"Rice-based: ..."` (unchanged label format)
  - `mode === 'tag'` → `"Tag: protein: fish: ..."`
  - `mode === 'component'` → `"Component #42: ..."`

### Form Layer (`src/components/rules/types.ts`, `RuleForm.tsx`)
- `MealTemplateFormState.selector` is a discriminated union with `mode: ''|'base'|'tag'|'component'`
- Four new `FormAction` variants: `SET_TEMPLATE_SELECTOR_MODE`, `SET_TEMPLATE_BASE_TYPE`, `SET_TEMPLATE_TAG_FILTER`, `SET_TEMPLATE_COMPONENT_ID`
- `formReducer` handles all four new actions
- `isFormValid` validates all selector modes
- `handleSave` builds `selector` object from form state
- `EXAMPLE_PRESETS['rice-lunch-dinner']` updated to use `selector: { mode: 'base', base_type: 'rice-based' }`

### UI (`src/components/rules/RuleFormFields/MealTemplateFields.tsx`)
- Level 1: RadioGroup with three options: Base type / Tag / Component
- Level 2a (Base): Shows rice-based/bread-based/other radio buttons
- Level 2b (Tag): Shows 4 Select dropdowns (dietary, protein, regional, occasion)
- Level 2c (Component): Shows component type Select + Combobox loaded via `useLiveQuery` from DB

### Generator (`src/services/generator.ts`)
- `selectorMatchesForSlotAssignment()`: Only base-mode rules apply to slot assignment
- `selectorMatches()`: Handles all three modes — base (base_type equality), tag (component tag intersection), component (slotComponentIds membership)
- `getMealTemplateAllowedSlots()` updated to use `selectorMatchesForSlotAssignment`
- `getApplicableMealTemplates()` updated with `baseComponent?` and `slotComponentIds?` parameters
- Two-pass matching in `generateWeekPlan`:
  - `baseTemplates`: after base selection, no slotComponentIds — matches base+tag modes for curry/subzi exclusion
  - `fullTemplates`: after base+curry+subzi selection, with full `slotComponentIds` — matches all modes for extra exclusion/requirement
