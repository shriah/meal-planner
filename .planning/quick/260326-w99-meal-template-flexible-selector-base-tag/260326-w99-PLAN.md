Current: Filters `r.base_type === baseType`.
New: Match based on selector mode:

Update the function signature:
```typescript

  baseType: BaseType,
  day: DayOfWeek,
  slot: MealSlot,
  mealTemplateRules: MealTemplateRule[],
  baseComponent?: ComponentRecord,
  slotComponentIds?: number[],
): MealTemplateRule[]
```

The matching logic per rule:
```typescript
function selectorMatches(
  selector: MealTemplateRule['selector'],
  baseType: BaseType,
  baseComponent?: ComponentRecord,
  slotComponentIds?: number[],
): boolean {
  switch (selector.mode) {
    case 'base':
      return selector.base_type === baseType;
    case 'tag': {
      if (!baseComponent) return false;
      const f = selector.filter;
      if (f.dietary_tag && !baseComponent.dietary_tags.includes(f.dietary_tag)) return false;
      if (f.protein_tag && baseComponent.protein_tag !== f.protein_tag) return false;
      if (f.regional_tag && !baseComponent.regional_tags.includes(f.regional_tag)) return false;
      if (f.occasion_tag && !baseComponent.occasion_tags.includes(f.occasion_tag)) return false;
      return true;
    }
    case 'component':
      return slotComponentIds?.includes(selector.component_id) ?? false;
  }
}
```

This two-pass approach is cleaner than trying to match everything at once. Store results in `baseTemplates` (first pass) and `fullTemplates` (second pass).
