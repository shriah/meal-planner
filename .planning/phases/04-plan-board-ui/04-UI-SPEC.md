## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gap inside component rows, lock icon padding |
| sm | 8px | Between component rows within a cell, tag badge gaps |
| md | 16px | Cell internal padding, sheet header padding, filter chip row gap |
| lg | 24px | Grid row gap, section padding in sheet |
| xl | 32px | Column header height, page horizontal padding (sm:) |
| 2xl | 48px | Empty state vertical padding |
| 3xl | 64px | Page top/bottom breathing room |

Exceptions:
- Touch targets for lock icons: minimum 44px tall click area (accessibility — source: WCAG 2.5.5)
- Day column width: fixed at 120px on desktop, full-width scrollable on mobile (horizontal scroll)

## Color

All values from `globals.css` CSS custom properties (olive palette, oklch).

| Role | Token | Usage |
|------|-------|-------|
| Dominant (60%) | `--background` oklch(1 0 0) | Page background, grid background |
| Secondary (30%) | `--card` oklch(1 0 0) / `--muted` oklch(0.966 0.005 106.5) | Cell backgrounds (unlocked: card; muted for row alternation), sheet surface |
| Accent (10%) | `--primary` oklch(0.852 0.199 91.936) — olive-yellow | Reserved for: Generate Plan button, Regenerate button, active lock icon (filled padlock), selected item in meal picker |
| Warning | `--chart-1` oklch(0.845 0.143 164.978) — amber tone | Violated slot border highlight, warning banner background tint |
| Destructive | `--destructive` oklch(0.577 0.245 27.325) | Not used directly in Phase 4 (no delete actions); reserved for future phases |

**Accent reserved for (explicit list):**
1. "Generate Plan" button (primary CTA, empty state)
2. "Regenerate" button (primary CTA, populated state)
3. Filled lock icon — when a component row is locked
4. Selected row highlight in meal picker sheet

**Locked cell visual distinction (PLAN-02 contract):**
- Locked component row: `bg-muted` background + filled `Lock` icon in `text-primary` color + `text-muted-foreground` for component name text
- Unlocked component row: `bg-card` background + outline `Unlock` icon in `text-muted-foreground` + `text-foreground` for component name text

**Warning slot highlight (from CONTEXT.md):**
- Affected cell: `border-2 border-amber-400` (or `border-chart-1`) on the cell wrapper
- Warning icon: `AlertTriangle` from lucide-react in amber color inside the cell, triggering a Tooltip with the specific warning message

## Component Inventory

### Grid layout

- **PlanBoard** (page root at `/`): full-width container, horizontal scroll on mobile
- **Grid structure**: CSS grid, 8 columns (1 row-label col + 7 day cols), 4 rows (1 header row + 3 slot rows)
  - Row labels column: fixed 80px wide, sticky left
  - Day columns: min 120px each, equal width
  - Slot rows: auto height, cells expand to fit all component rows
- **DayColumnHeader**: day name (Mon–Sun) + "Lock Day" button (outline variant, small)
  - "Lock Day" locks all 4 component locks across all 3 slots for that day (12 locks total)
- **SlotRowLabel**: "Breakfast" / "Lunch" / "Dinner" — sticky left, aligned center

### Cell component

- **MealCell** (one per day×slot): rounded-md border, padding md (16px)
  - Contains 4 **ComponentRow** sub-rows: base, curry, subzi, extras
  - Empty slot (before generation): single dash "—" centered in cell
- **ComponentRow**: horizontal flex row
  - Left: lock icon button (44px touch target, lucide Lock/Unlock)
  - Right: component name text (clickable if unlocked, triggers meal picker)
  - Locked state: `bg-muted`, filled lock icon in `text-primary`, name in `text-muted-foreground`, not clickable
  - Unlocked state: `bg-card`, outline unlock icon in `text-muted-foreground`, name in `text-foreground`, hover shows `cursor-pointer` and subtle `bg-accent/10` highlight

### Meal picker sheet

- **MealPickerSheet** (shadcn Sheet, side="bottom"): opens when unlocked component row is clicked
  - Sheet title: "Pick [Component Type]" (e.g., "Pick Curry")
  - Top section: text search Input + dietary/regional tag filter chips (same Badge pattern as ComponentTab)
  - List: scrollable, each item shows component name + dietary & regional tag badges + slot eligibility note
  - Selecting an item: replaces that component in the slot, closes the sheet, auto-saves to Dexie
  - Close: X button in sheet header (default Sheet behavior)

### Action bar

- **PlanActionBar**: sticky bottom bar (or top, at Claude's discretion — top recommended for desktop)
  - "Regenerate" button: primary variant, accent color, `RefreshCw` icon prefix
  - Generator running state: button shows spinner (`animate-spin` on `RefreshCw`), text "Generating..."

### Warning banner

- **WarningBanner**: dismissable, amber tinted, appears above the grid
  - Shows count summary: "2 slots could not be filled — see highlighted cells"
  - Dismiss button: X icon, clears banner (warnings still visible on cells until next regeneration)
  - Auto-clears on next successful regeneration with no violations

### Navigation

- **AppNav**: header with app name and nav links to `/library` and `/settings/slots`
  - Matches the layout pattern established in Phase 2 (currently no shared nav — Phase 4 adds it)

## Interaction States

| State | Trigger | Visual |
|-------|---------|--------|
| Empty (no plan) | First visit, no plan in Dexie | 7×3 grid with "—" in all cells + prominent "Generate Plan" button centered above grid |
| Generating | Generate/Regenerate pressed | Button spinner, grid cells of unlocked slots show skeleton pulse |
| Populated | Generation complete | Full cell content rendered |
| Locked component | Lock icon clicked | Row background → muted, icon → filled Lock in primary, text → muted-foreground |
| Day locked | "Lock Day" column header button | All 12 component rows in that column enter locked state simultaneously |
| Picker open | Unlocked component name clicked | Sheet slides up from bottom, pre-filtered to component type |
| Warning | Generator returns Warning[] | Amber banner + amber border on affected cells + AlertTriangle icon with Tooltip |
| Warning dismissed | X on banner | Banner hides; cell highlights remain until next regeneration |
