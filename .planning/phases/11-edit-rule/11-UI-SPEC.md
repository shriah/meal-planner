## Interaction Contract

### Edit Button Placement

- Location: `RuleRow` action area, between the enable/disable toggle and the delete (Trash2) button.
- Icon: `Pencil` (lucide-react, h-4 w-4).
- Button: `variant="ghost"` `size="icon"` — matches existing Trash2 delete button style exactly.
- Tooltip: "Edit rule" (matches "Delete rule" tooltip pattern).
- `aria-label="Edit rule"`.

### Edit Sheet Layout

- Side: `right` (full-height side panel — matches shadcn Sheet default for form-heavy content, distinct from MealPickerSheet which uses `bottom`).
- Width: `sm:max-w-sm` (inherited from SheetContent default — approx 384px on desktop, 75vw on mobile).
- Structure:
  1. `SheetHeader` → `SheetTitle` "Edit Rule" + `SheetDescription` "Changes are saved when you click Save Rule."
  2. Scrollable body: rule name `Input`, `RuleFields` (reused), `RuleImpactPreview` (reused).
  3. `SheetFooter` (pinned bottom, `mt-auto`): "Save Rule" button (primary, disabled while saving or invalid) + "Discard Changes" button (ghost, closes sheet without saving).
- The built-in `SheetContent` close button (X, top-right) remains visible and also discards edits (EDIT-04).

### Pre-population (EDIT-02)

- On sheet open, dispatch `LOAD_PRESET` action with a `RuleFormState` derived from the existing `RuleRecord.compiled_filter` and `rule.name`.
- A `decompileRule(compiled_filter): RuleFormState` pure function must be implemented to reverse the compile step.
- All fields (name, target mode, days, slots, selection, allowed_slots, skip_component_types, extra categories) must be populated — no field should fall back to its initial empty/default value.

### Save Behavior (EDIT-03)

- On "Save Rule": call `updateRule(rule.id, { name, compiled_filter, enabled: rule.enabled })` — preserves `enabled` state and `created_at`.
- No navigation away — sheet closes after successful save, user stays on `/rules`.
- Button label: "Save Rule" (idle) → "Saving..." (in-flight).
- Disabled when: form is invalid OR save is in-flight.
- On save failure: display a toast with copy "Failed to save rule. Please try again." Sheet stays open, button re-enables so the user can retry.

### Cancel / Close Behavior (EDIT-04)

- "Discard Changes" button, X close button, and Escape key all close the sheet without calling `updateRule`.
- Sheet local state is re-initialized from the original `rule` prop each time the sheet opens (controlled via `open` prop reset).

### State Management

- Edit sheet open state managed by `RuleRow` local `useState<boolean>`.
- Form state managed by `useReducer(formReducer, ...)` inside the edit sheet component — same reducer as `RuleForm`.
- Initial state seeded from rule prop on mount via `useEffect` with `LOAD_PRESET` dispatch.
