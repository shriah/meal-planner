## Color

All screen values from CSS custom properties in globals.css (Source: globals.css).

| Role | CSS Token | OKLCH (light) | Usage |
|------|-----------|---------------|-------|
| Dominant (60%) | --background | oklch(1 0 0) | Page background, plan grid background |
| Secondary (30%) | --card / --muted | oklch(1 0 0) / oklch(0.966 0.005 106.5) | Week navigator background, read-only banner background, meal cells |
| Accent (10%) | --primary | oklch(0.852 0.199 91.936) — olive-yellow | Export button only; week navigator "current week" indicator dot |
| Destructive | --destructive | oklch(0.577 0.245 27.325) — red | Not used in this phase |
| Muted text | --muted-foreground | oklch(0.58 0.031 107.3) | Read-only banner text, past week date label, navigator chevron |
| Border | --border | oklch(0.93 0.007 106.5) | Week navigator divider, read-only banner border |

**Accent reserved for:** Export button (primary variant), current-week indicator. NOT used for headings, nav links, or the week navigator arrows.

**Read-only past week visual treatment (D-07 from CONTEXT.md):** Apply a muted amber info banner directly below the week navigator: `border border-amber-300 bg-amber-50 text-amber-900 rounded-md px-4 py-2 text-sm`. The banner reads: "This is a past week — the plan is read-only." No additional color muting of the grid itself; the banner is the sole read-only indicator.

**Dark mode:** All tokens have dark-mode values in globals.css. No additional dark-mode overrides needed.

**PNG export color palette (satori template — not screen CSS):**

These are hex approximations of the app's OKLCH tokens, chosen for WhatsApp-shareable legibility. Derived from globals.css tokens as directed by CONTEXT.md D-13.

## Interaction Contracts

### Week navigation

| Action | Behavior |
|--------|----------|
| Click ChevronLeft | `navigateToWeek(subWeeks(currentWeekStart, 1))`. Load week record from Dexie via `getWeekPlan(newWeekStart)`. If no record exists and week is in the past → show empty grid (read-only). |
| Click ChevronRight | `navigateToWeek(addWeeks(currentWeekStart, 1))`. If week is in the future and no record exists → show future empty state with "Generate Plan for This Week". |
| Navigate to current week | `navigateToWeek(startOfCurrentWeek())`. Loads active plan state. |

**isReadOnly derivation:** `currentWeekStart < startOfISOWeek(today)`. Pure date comparison, no server calls.

**Auto-save trigger:** Any plan mutation (lock, swap, regenerate) calls `saveWeekPlan(currentWeekStart, plan, locks)` fire-and-forget. Same pattern as existing `saveActivePlan`.

### Export flow

| Step | Behavior |
|------|----------|
| Click "Export PNG" | Set `isExporting = true`. POST to `/api/export-plan` with serialized `WeeklyPlan` + resolved component names. |
| Desktop | Route Handler responds with `Content-Disposition: attachment; filename="meal-plan-{weekLabel}.png"`. Browser triggers download. |
| Mobile (Web Share API available) | `navigator.share({ files: [pngBlob] })`. Native share sheet opens. |
| Mobile (Web Share API unavailable) | Fall back to download link. |
| Error | Set `isExporting = false`. Show inline error: "Export failed. Try again." below the Export button — plain `<p className="text-sm text-destructive">` (no toast, no modal). |

Detection: `if (navigator.canShare && navigator.canShare({ files: [testBlob] }))` before calling `navigator.share`.

### Plan grid read-only mode (past weeks)

When `isReadOnly === true`:
- Lock toggle buttons are hidden (`hidden` class, not disabled)
- MealPickerSheet does not open on cell click
- Regenerate button is hidden from PlanActionBar
- Read-only banner is visible

When `isReadOnly === false` (current and future weeks):
- All controls visible and interactive as in Phase 4

### Zustand store changes

| New state | Type | Initial value |
|-----------|------|---------------|
| `currentWeekStart` | `string` (ISO date `"YYYY-MM-DD"`) | ISO date of current week's Monday |
| `isReadOnly` | `boolean` | `false` |
| `isExporting` | `boolean` | `false` |
| `exportError` | `string \| null` | `null` |

New actions: `navigateToWeek(weekStart: string)`, `exportPlan()`.

## Accessibility Contract

| Pattern | Requirement |
|---------|------------|
| Week navigator prev/next | `aria-label="Previous week"` and `aria-label="Next week"` on Button elements |
| Week label | `aria-live="polite"` on the week label span so screen readers announce navigation changes |
| Export button | Descriptive label "Export PNG" sufficient; no additional aria needed |
| Read-only banner | `role="status"` on the Alert so it is announced to screen readers when it appears |
| PNG export | No accessibility requirements — image is for sharing; no alt text contract needed |
| Color contrast — PNG | All breakfast/lunch/dinner text on their respective backgrounds meets WCAG AA (verified by color choices: amber-900 on amber-100, green-900 on green-100, blue-900 on blue-100) |
