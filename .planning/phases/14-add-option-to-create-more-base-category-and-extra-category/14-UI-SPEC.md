## Interaction Rules

### Entry Point

Use a dedicated `Manage Categories` secondary action in the Library page header, aligned opposite the `Component Library` title in [`src/components/library/ComponentLibrary.tsx`](/Users/harish/workspace/food-planner/src/components/library/ComponentLibrary.tsx). Do not place category creation inline inside [`src/components/library/ComponentForm.tsx`](/Users/harish/workspace/food-planner/src/components/library/ComponentForm.tsx).

### Management Surface

Open a `Sheet` for category management to stay consistent with the existing rule-edit pattern and installed primitives. The sheet contains exactly two sections or tabs:

- `Base Categories`
- `Extra Categories`

Visual priority must read in this order:

1. Active section title or active tab
2. Top `Add Category` control for that section
3. Category rows beneath it

The active section header plus its `Add Category` control is the focal point on first open. Category rows are supporting management content and should not compete visually with the primary action.

Each section includes:

- A one-line explanation of where the category is used.
- A single `Add Category` control at the top of the list.
- A scrollable list of existing categories sorted by current display name.

### Category Row Contract

Each category row is a simple management row, not a card. Use:

- 48px minimum row height.
- Name on the left.
- Optional small usage summary on the right in muted text, if available.
- `Rename` and `Delete` actions aligned to the row end.

Only one row may enter rename mode at a time. Rename mode swaps the static name for a single-line input plus `Save Name` and `Stop Renaming`. Hitting `Escape` triggers `Stop Renaming` with no save. Hitting `Enter` saves when valid.

### Add Flow

`Add Category` inserts a new inline draft row at the top of the active kind list and autofocuses the name input. Keep the draft lightweight:

- Empty by default.
- Save disabled until trimmed name is non-empty and unique within that kind.
- `Discard Draft Category` removes the draft row entirely.

Do not close the sheet after a successful add. Keep the user in context so they can add multiple categories in sequence.

### Delete Flow

Delete is allowed even when the category is referenced. Clicking `Delete` reveals an inline destructive confirmation strip directly under that row, following the existing destructive-alert pattern used in [`src/components/library/DeleteConfirmStrip.tsx`](/Users/harish/workspace/food-planner/src/components/library/DeleteConfirmStrip.tsx).

The confirmation must state that cleanup is automatic. It must not ask the user to manually fix references before proceeding.

### Dynamic Option Propagation

Every surface that currently lists base types or extra categories must render live category-backed options:

- Base component create/edit selects
- Extra category select
- Compatible base checklist for extras
- Rule target base-category options
- `require_extra` category options

Keep the current extra compatibility interaction as a multi-select checklist. Do not replace it with a combobox or token input.

### Rename and Delete Semantics in UI

Rename is identity-preserving. The UI should update visible labels everywhere without showing an intermediate stale name after save.

Delete cleanup is system-owned. The UI contract is:

- Do not block delete because of existing usage.
- Do show that the category is in use if usage data is available.
- Do not leave orphaned labels or raw IDs visible after delete.
- If a deleted category causes a rule to become invalid/disabled, the rule list must show the normalized post-delete result, not an error shell.

### Feedback and State Handling

Use the global toaster already mounted in [`src/app/layout.tsx`](/Users/harish/workspace/food-planner/src/app/layout.tsx) for save/delete failures. Keep success feedback quiet unless the action meaningfully changes the current view; default to optimistic list refresh via live query rather than celebratory toasts.

Loading contract:

- Show the sheet chrome immediately.
- If categories are still loading, show 3-5 muted skeleton-like placeholder rows or static bordered placeholders.
- Do not block the whole library page behind a spinner.
